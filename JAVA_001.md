JAVA의 특징 및 장단점

1. JAVA의 특징(겸 장점인거 같기도…)
    
    → why? 다른 언어에 비해 이런 부분들을 체계적으로 구축할 수 있다는 부분이 자바의 특징이자 장점인거 같다.
    
    - 객체지향 언어
        
        → 캡슐화, 상속성, 다형성을 지원하는 언어
        
        → 객체를 만들기 위해 클래스를 작성하고 객체들끼리 연결하여 목적에 맞는 프로그램을 만들 수 있다.
        
        → 객체 지향 언어라 유지보수가 쉽고 확장성이 좋음
        
    - 플랫폼에 영향을 받지않고 다양한 환경에서 사용 가능
        
        → JVM이 현재 이용중인 OS에 맞게 번역하여 프로그램을 돌린다.
        
    - 오픈소스 라이브러리와 활발한 커뮤니티로 다양한 프로그램을 제작하기 좋다.
    - 안정적이다.
        
        → C 친구들의 포인터를 사용하지 않기에 메모리를 직접 제어하지는 않지만 직접 제어에서 발생하는 오류가 없어 훨씬 안정적이다.
        
2. JAVA의 장단점
    - 단점
        - 속도가 느린편이다.
            
            → 다른 언어에 비해 코드를 기계어로 변환하기 위한 작업도 필요함.
            
        - GUI부분에서 다른 프레임워크를 활용하더라도 복잡한 GUI를 개발하기 충분하지 않다. (→ 충분히 개선할 수 있는 부분이라고 생각함. 요즘 잘나와서)
        - 요금 지불 라이센스
            
            → 오라클을 활용하기 위해(학습이 아닌 실무 차원에서) 수수료를 내야한다.
            
        - 메모리 사용량이 많다.
        - 너무 많은 라이브러리와 오픈소스로 어려울 수 있다.(사실 쓰다보면 그때그때 찾아보면서 익혀도 될거 같기는 함, 업데이트 되는것도 확인해야하고 개발자는 늘 공부해야하니까)
